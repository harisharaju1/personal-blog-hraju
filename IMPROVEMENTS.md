# Blog Improvements

Enhancements shipped to align with the "Lessons Learned" blog strategy —
architecture proof-of-work, social hooks, and owned content.

---

## 1. Syntax-highlighted code blocks

**Files:** `lib/markdown.ts`, `components/posts/post-body.tsx`, `app/globals.css`

Replaced `react-markdown` with a `unified` server-side pipeline:
`remark-parse → remark-gfm → remark-rehype → rehype-pretty-code → rehype-stringify`.

- Uses [rehype-pretty-code](https://rehype-pretty-code.netlify.app/) backed by [Shiki](https://shiki.style/)
- Dual theme: `github-light` in light mode, `github-dark-dimmed` in dark mode
- Switching is handled purely by CSS — no hydration flash
- `PostBody` is now an `async` server component; no client JS needed for rendering

The `@tailwindcss/typography` plugin provides the `prose prose-neutral dark:prose-invert` 
wrapper so all markdown elements (headings, blockquotes, tables, lists) are styled automatically.

**New dependency added:** `unified`, `remark-parse`, `remark-rehype`, `rehype-pretty-code`,
`rehype-stringify`, `@tailwindcss/typography`  
**Removed:** `react-markdown`

## 2. Copy button on code blocks

**File:** `components/posts/code-copy-buttons.tsx`

A client component that mounts after hydration and injects a "Copy" button into every `<pre>`
block inside `#post-body`. Appears on hover, copies raw code text to clipboard, confirms
with "Copied!" for 2 seconds.

Implemented with DOM manipulation + inline styles to avoid Tailwind purging dynamically
injected class names.

---

## 3. Open Graph / social preview meta tags

**Files:** `app/layout.tsx`, `app/posts/[id]/page.tsx`, `lib/config.ts`

Every post page now exports `generateMetadata` that populates:
- `<title>Post Title — hraju.dev</title>`
- `og:title`, `og:description`, `og:type: article`, `og:publishedTime`, `og:url`
- `twitter:card: summary`, `twitter:title`, `twitter:description`
- `<link rel="canonical">` per post

The root layout sets `metadataBase`, `title.template`, and site-wide OG defaults.
Post descriptions are auto-generated by stripping markdown from the first 160 chars of the body.

**New file:** `lib/config.ts` — `SITE_URL`, `SITE_TITLE`, `SITE_AUTHOR`, `SITE_DESCRIPTION`

## 4. RSS feed

**File:** `app/feed.xml/route.ts`

`GET /feed.xml` returns a valid RSS 2.0 feed of all posts, newest first. Each item includes
title, URL, GUID, plain-text description, and publication date. The feed URL is advertised
in the `<head>` via `alternates.types` in the root layout metadata.

XML is escaped correctly. Response is cached for 1 hour (`s-maxage=3600`).

## 5. Read time estimate

**Files:** `lib/text.ts`, `components/posts/post-card.tsx`, `app/posts/[id]/page.tsx`

`readingTime(body)` counts words and returns `"N min read"` at 200 wpm (minimum 1 minute).
Shown on both the post list cards and the individual post header.

## 6. Post excerpts on the home page

**Files:** `lib/text.ts`, `components/posts/post-card.tsx`

`excerpt(body, 140)` strips markdown syntax (code blocks, links, images, formatting chars)
and returns the first 140 characters of plain text. Rendered as a muted subtitle under each
post title. Cards also use a nicer date format (`May 1, 2026`).

**Files:** `playwright.config.ts`, `e2e/navigation.spec.ts`

19 end-to-end tests across two suites:

**Site structure** — page titles, header links (About, RSS, logo), navigation between home and about, 404 for unknown routes.

## 13. AI-navigable routes

### `llms.txt` — `app/llms.txt/route.ts`

`GET /llms.txt` follows the [llmstxt.org](https://llmstxt.org) spec: an H1, a blockquote description, then a `## Posts` section listing every post with its URL and a plain-text excerpt. Includes a `## Notes for LLMs` section explaining authorship and content intent. This is the index file AI agents and crawlers use to understand the site.

### `llms-full.txt` — `app/llms-full.txt/route.ts`

`GET /llms-full.txt` returns the raw markdown body of every post, each with URL, author, and publication date. Intended for RAG (retrieval-augmented generation) indexing — an LLM can fetch this once and have the full text of every post without crawling individual pages.

### `robots.txt` — `app/robots.ts`

`GET /robots.txt` allows all crawlers at `/` and disallows `/admin/`. Advertises the sitemap URL. Generated via the Next.js built-in `MetadataRoute.Robots` API.

### JSON-LD structured data — `app/posts/[id]/page.tsx`

Every post page now includes a `<script type="application/ld+json">` block with `schema.org/BlogPosting` markup: headline, description, author (with `/about` URL), `datePublished`, and canonical URL. This is what search engines, AI assistants, and scrapers use to understand individual posts.
